Case Management Board

Es una herramienta de visualización, manejo y gestion de Casos para una firma de abogados. Tiene diversas opciones personalizadas para el manejo de casos que viene a remplazar la anterior tabla que se tenia en Laravel 5 la cual se limitaba a unos cuantos filtros y una descarga. Algunas de las funcionalidades son:

  • - Creación y asignación de Tags
  • - Creación y asignación de Status y Meta Status personalizados
  • - Asignación de prioridades
  • - Creacion de Tabs(vistas) por usuario
  • - Vista de Tabla y vista de Columns(cards)
  • - Customizar tanto el orden de las columnas como la seleccion de columnas visibles por vista
  • - La posibilidad de agrupar y filtrar(colocar más de un filtro)
Es una herramienta que esta basada en la plataforma de organización "Infinity"

Tecnologias utilizadas:

  • VueJS - Vue3 - integrado como estático en Laravel 5
  • Ant Design Vue - Libreria de VueJS
  • Node JS - GraphQL
  • MySQL
  • MongoDB

Front Project Manager

La plataforma Prima Law estuvo durante muchos años construida en PHP en una versión antigua de Laravel(la version 5) lo cual ponía muchas limitantes a la hora de hacer crecer la aplicación y hacerla cada vez más robusta y con un desarrollo horizontal. También al haber pasado por muchas manos y no haberse procurado buenas practicas y estructura, provocó que los problemas fueran cada vez más dificiles de mitigar y controlar. Por todas estas razones se decidió hacer una migración total del Front de Laravel a VueJS(Vue3), proyecto del cual quede a cargo de un grupo de 5 developers para sacarlo adelante.

Para esto se establecieron objetivos y lineamientos importantes:

  • - El nuevo Front tiene que tener una navegación fluída y reactiva entre cada una de sus secciones, evitar recargas de pagina.
  • - La utilización de Pinia,librería de Vue que te permite compartir estado entre componentes. Esto para asegurarse de que siempre que se navegue entre componentes no se esten trayendo datos innecesariamente, ahorrando uso de memoria(uno de los problemas principales que se tenia).
  • - Procurar una buena estructura y conexión entre componentes del proyecto.
  • - Reactividad entre usuarios. Para esto se implemento una estructura de websockets que permite la comunicación entre sesiones.

El seguimiento se hizo utilizando una metodología muy parecida a la de SCRUM o Ágil, estableciendo Sprints, una segmentación de tareas, los tiempos de cada tarea y quienes se dedicarían a cumplirlo. Reuniones diarias de entre 10 a 30 minutos con el equipo de front para ver los avances, los problemas, las posibles soluciones y cualquier otro tipo de novedad que se presente. Esos avances a su vez los presentaba en la reunion diaria general de todo el equipo de devs.

Las herramientas que se utilizaron para esto fueron:

  • - Spreadsheets(con Google Sheets), con varias vistas que identificaban cada sección de la aplicación a migrar a Vue. Un spreadsheet para el desarrollo, otro para las pruebas en staging y otro para pruebas finales y afinamiento en producción
  • - Infinity, para el seguimiento mas general de las tareas. También para una visualización más sencilla para el Project Manager de todos los devs y el Jefe de la aplicación.
  • - Notion, herramienta que se utiliza en el equipo para la organización de la documentación del desarrollo de la aplicación

Tecnologias utilizadas:

  • VueJS - Vue3 - integrado como estático en Laravel 5
  • Sass
  • Ant Design Vue - Libreria de VueJS
  • Pinia - Libreria de VueJS para la gestion de estados en la aplicación
  • Bootstrap
  • Node JS - GraphQL
  • MySQL
  • MongoDB
