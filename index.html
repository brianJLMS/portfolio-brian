Case Management Board

It is a case management and visualization tool for a law firm. It has several customized options for managing cases, replacing the previous table in Laravel 5, which was limited to just a few filters and a download option. Some of its functionalities are:

  • - Creation and assignment of Tags
  • - Creation and assignment of custom Statuses and Meta Statuses
  • - Assignment of priorities
  • - Creation of Tabs (views) per user
  • - Table view and Column (cards) view
  • - Customize both the column order and the selection of visible columns per view
  • - The ability to group and filter (apply more than one filter)
It is a tool based on the "Infinity" organization platform.

  • Typescript

  • MongoDB

Front Project Manager

The Prima Law platform was built for many years in PHP using an old version of Laravel (version 5), which created many limitations when trying to grow the application and make it more robust with horizontal development. Additionally, since it had passed through many hands without ensuring good practices and structure, the problems became increasingly difficult to mitigate and control. For all these reasons, it was decided to completely migrate the Front from Laravel to VueJS (Vue3), a project for which I was in charge of a team of 5 developers to move it forward.

To achieve this, important goals and guidelines were established:

  • - The new Front should have smooth and reactive navigation between its sections, avoiding page reloads.
  • - The use of Pinia, a Vue library that allows you to share state between components. This ensures that when navigating between components, unnecessary data is not fetched, saving memory (one of the main issues we had).
  • - Ensure good structure and connection between the project’s components.
  • - User interactivity. For this, a WebSocket structure was implemented to allow communication between sessions.

The project was tracked using a methodology similar to SCRUM or Agile, establishing Sprints, segmenting tasks, defining timelines for each task, and determining who would be responsible for them. Daily meetings of 10 to 30 minutes were held with the front-end team to review progress, problems, possible solutions, and any other updates. These updates were then presented at the general daily meeting with the entire development team.

The tools used for this were:

  • - Spreadsheets (using Google Sheets), with several views identifying each section of the application to migrate to Vue. One spreadsheet for development, another for testing in staging, and another for final testing and fine-tuning in production.
  • - Infinity, for general task tracking and a simpler view for the Project Manager and the Head of the application.
  • - Notion, a tool used by the team for organizing the application development documentation.

Technologies used:

  • VueJS - Vue3 - statically integrated into Laravel 5
  • Sass
  • Typescript
  • Ant Design Vue - VueJS library
  • Pinia - VueJS library for state management in the application
  • Bootstrap
  • Node JS - GraphQL
  • MySQL
  • MongoDB
  • Docker
